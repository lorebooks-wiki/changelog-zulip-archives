Andrew O'Brien (May 10 2025 at 15:01):

Re, CDN testing: I didn’t get to actually implementing this, but years ago when I want to test how well things like expiring a cached object, change propagation, or edge functions worked in a CDN based system I was evaluating, I looked long at Jepsen.

It’s most known for testing distributed databases and I’ve never seen anyone use it to test a CDN, but IIRC it has a lot of things you’d want baked in like checking to see how long it takes for a write to be reflected in reads and what happens when you try to write while another one is still propagated.

Also, it’s really easy to spin up worldwide distributed test nodes through AWS Marketplace.
